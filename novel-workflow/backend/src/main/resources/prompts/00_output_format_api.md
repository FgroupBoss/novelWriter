# API 输出格式（全阶段通用 · 静态前缀）

以下规则对所有阶段相同，便于 LLM 提供商（如 DeepSeek）缓存 prompt 前缀。

## 输出格式（必须严格遵守）

你必须且只能使用 **FILE 块** 输出，每个目标文件一块。块格式：

```
===FILE:{相对路径}===
（此处写入该文件的完整 Markdown 内容）
===END===
```

## 通用规则

1. 每个文件内容必须是完整、可落盘的 Markdown。
2. 不要在 FILE 块之外输出大段说明；简要说明可写在第一个文件开头的 HTML 注释内。
3. 不要遗漏「当前任务」中列出的任何一个输出文件。
4. 输出文件的路径、数量与顺序以用户消息末尾「当前任务」章节为准。
